# シグナル検証プロトコル（プレレジストレーション）

**固定日: 2026-06-11（signal_history に最初の確定レコードが入る前に固定）**

本書はシグナル検証の合格基準を**結果を見る前に**固定する事前登録文書である。
基準の事後変更は禁止。変更が必要になった場合は本書を書き換えず、
新版を追記して旧版を残す（何をいつ変えたかを監査可能にする）。

---

## 1. 対象サンプル

- `signal_pending_4h.jsonl` / `signal_pending_1d.jsonl` の**全記録**
  （= verify-signals が確定させて `signal_history*.jsonl` に追記したもの全件）。
- 記録基盤は PR#6（2026-06-10）以降にのみ存在するため、**全件がクリーンなサンプル**
  （それ以前のデータは不存在。選別・除外による生存バイアスの余地がない）。
- 集計時の重複排除は `signal_id` で行う。

## 2. 検証軸

- **主検証軸: 168h（1週間）**。副軸: 72h（3日）。
- 1h / 24h は**織り込み速度の参考のみ**とし、合否判定に使わない。
- 方向一致・%リターンの定義は記録時点のもの（bearish は符号反転済み、
  「シグナル方向が正しければ正のリターン」）。direction=neutral は方向一致率の母数から除外。

## 3. バケット定義（|conviction_score| ベース・互いに排他）

| バケット | 定義 |
|---|---|
| divergence | `is_divergence=true`（**別枠**。下の3バケットには入れない） |
| critical | \|conviction\| >= 60 |
| normal | 30 <= \|conviction\| < 60 |
| reference | 15 <= \|conviction\| < 30 |
| (other) | \|conviction\| < 15（--no-filter 記録等。参考表示のみ・判定対象外） |

※ other は定義上存在するが、**通常 run は silent（\|conviction\|<15・非divergence）を記録しないため常に n=0 が正常**（`--no-filter` の手動実行時のみ入りうる。空でもバグではない）。

## 4. 指標

- **方向一致率**: hit数 / 方向性あり記録数。**Wilson 95% 信頼区間**を併記
  （z=1.96、自前実装・scipy 不使用）。
- **平均符号付き%リターン** と **中央値**。
- **ロバスト性検査**: |リターン| の上位5%を winsorize（95パーセンタイルで丸める）した
  平均を併記する。生値と winsorize 値が大きく乖離する場合、ロール跨ぎ・ギャップ等の
  少数レコード汚染を疑い、該当レコードを個別検査する（基準には影響させない。検査トリガーのみ）。

## 5. 合格基準（事前固定）

**主判定は 168h・バケットは divergence を除く3つ:**

- **(a) 単調性**: 168h の方向一致率が `reference < normal < critical` の順に単調増加していること。
  （確信度スコアが「強いほど当たる」設計意図どおりに機能しているかの検証）
- **(b) critical の実力**: critical バケットの方向一致率の **Wilson 95%CI 下限が 50% を超える**こと。
  （ノイズではなく統計的に方向性があることの最低条件）
- **(c) サンプルサイズ**: 本判定は**各バケット n >= 100** に達してから行う。
  n >= 30 の段階での集計は「**interim**」と明記し、**判断材料にしない**
  （途中経過を見て基準や実装を調整することを禁じる。それをやればプレレジの意味がなくなる）。

### divergence バケット（別枠の検証）

divergence は「TAとニュースが逆を向いた」記録なので、**どちらが正しかったか**が本題:
- **TA方向の一致率**（記録された direction 基準 = 既存の dir_hit）
- **ニュース方向の一致率**（sign(net_direction) と生リターンの符号一致）
を両方算出する。ニュース方向の一致率が有意に高ければ「divergence はニュース側に
賭けるシグナル」として扱う再設計の根拠になる。判定の n 基準は同じ（n>=100、interim は n>=30）。

## 6. 不合格時の帰結（事前固定）

- (a) 単調性が崩れる、または (b) critical CI下限が50%以下の場合:
  **TA確信度を「方向ゲート」から「タイミング参考」に格下げ**し、閾値（15/30/60）と
  ルーティング設計を再設計する。方向判断の主軸は再設計後の体系（ニュース整合性・
  将来のshadowファクター）に移す。
- 部分的な不合格（例: critical のみ合格）の場合は、合格バケットのみを通知閾値として残し、
  不合格バケットは記録のみに格下げする。

## 7. 新ファクターの shadow → 昇格ルール（CLAUDE.md 原則③の運用詳細）

1. 新ファクター（DXY/実質金利、CoT、イベントゲート、限月構造、流動性等）は
   **必ず shadow（記録のみ・conviction 不適用）で開始**する。レコードに新フィールドとして
   追加し（スキーマは追加のみ）、既存の数値経路には触れない。
2. 昇格判定は本プロトコルと同じ枠組みで行う: shadow フィールドの有無/値で層別した
   168h 方向一致率を比較し、**n >= 100 かつ改善が Wilson CI で重なっていない**ことを
   昇格の必要条件とする。
3. 昇格時も一度に1ファクターのみ（同時昇格は寄与の切り分けを壊す）。

## 8. 集計の実行

- `python scripts/report_validation.py` （読み取り専用・本体パイプライン無改変・stdlib のみ）。
- history は月次アーカイブ構造（`signal_history.jsonl` + `signal_history_YYYYMM.jsonl`）のため
  glob で全ファイルを読む。
- データ不足のセルは n 表示つきの空欄とし、エラーにしない。

## 9. macro 昇格テスト定義（shadowデータ発生前に固定）

- 対象: `macro` フィールドが non-null のシグナル全件（`real_yield_source` はソース別の
  層別が可能だが、主判定はソース混合で行う）。
- **順方向群**: conviction 方向と macro レジームが整合するシグナル
  （bullish かつ `regime="ry-1_dxy-1"`、または bearish かつ `regime="ry+1_dxy+1"`）。
- **逆方向群**: conviction 方向と macro レジームが逆
  （bullish かつ `ry+1_dxy+1`、または bearish かつ `ry-1_dxy-1`）。
- **判定**: 両群の 168h 方向一致率を比較。**各群 n ≥ 100**、
  **Wilson 95%CI が重ならない改善**（順方向群の CI 下限 > 逆方向群の CI 上限）を
  昇格条件とする。
- 中間レジーム（trend=0 を含む regime）はどちらの群にも入れない。
- 昇格した場合の適用方法（係数への織り込み方）は昇格判定の後に別途設計する
  （本書はテスト定義のみを固定する）。

参考（Stage1 オフライン検証 2026-06-10, research/macro_offline_report.md）:
事前ルールは3サブ期間の符号不安定 [-0.46, +0.14, +0.95] で不成立。ただし直近ほど
改善する単調変化があり、shadow 実データでの再判定に委ねる（この参考情報は
判定基準ではない）。

## 10. ニュース層帰属テスト（初回確定データ発生前に固定）

- **対象**: routing=divergence（`is_divergence=true`）を**除く**全レコード。
- **normalized の導出（記録スキーマに直接保存されていないため式を固定する）**:
  `normalized = sign(ta_score) × net_direction`
  （ta_score / net_direction とも記録済みフィールド。ta_score=0 は sign=0 → normalized=0）
- **群分け**:
  | 群 | 定義 |
  |---|---|
  | 増幅群 | news_count > 0 かつ normalized ≥ +0.2 |
  | 減衰群 | news_count > 0 かつ normalized ≤ -0.2 |
  | 中立群 | news_count > 0 かつ -0.2 < normalized < +0.2 |
  | 無風 | news_count = 0（**中立に含めず別掲**。クレジット枯渇期間の記録も自動的にここへ入る） |
- **事前予測**: 168h 方向一致率が **減衰 < 中立 < 増幅** の単調性
  （ニュース係数が確信度を正しく増減させているかの直接検証）。
- **判定**: 本判定は各群 **n ≥ 100**・Wilson 95%CI 付き。n ≥ 30 は interim 明記で
  判断材料にしない（v1 §5c と同じ規律）。
- 集計は `scripts/report_validation.py` に同梱（読み取り専用のまま）。
